Автоматизація деплою та SEO-налаштувань у WordPress

Ніша / Проєкт: Веб-портали на WordPress

Інструментарій: WordPress REST API, MCP (Model Context Protocol), Python (requests), RankMath

Кейс показує, як позбутися рутини при наповненні сайтів. Побудовано пайплайн для масової публікації статей, обходу захисту WAF та автоматичного налаштування всіх необхідних SEO-метаданих.

Етап 1: Обхід WAF та налаштування REST API

При масовій публікації серверні захисти (Cloudflare, ModSecurity) часто блокують API-запити. Я вирішив цю проблему:

  • Налаштування Application Passwords у WordPress.
  • Імплементація ротації кастомних User-Agent'ів та кастомних Headers у Python-скриптах для безперебійного проходження через WAF.

Етап 2: Масова автоматична публікація

Процес публікації повністю автоматизовано за допомогою спеціальних MCP інструментів та скриптів:

Gutenberg Ready: Скрипти коректно форматують контент у блоки Gutenberg, зберігаючи стилі, таблиці, та шорткоди без пошкодження HTML-структури сайту.
  • Автоматичне завантаження медіафайлів (зображень) та генерація для них оптимізованих `alt` атрибутів.
  • Прив'язка статей до відповідних категорій та тегів (Taxonomy mapping).

Етап 3: Автоматизація RankMath та Schema.org

Публікація тексту — це лише половина справи. Пайплайн також автоматизує SEO-параметри:

  • Динамічна генерація та інсерт Focus Keywords, SEO Titles та Meta Descriptions через мета-поля RankMath.
  • Автоматична розмітка JSON-LD (FAQ, Article, Breadcrumbs), що одразу генерує rich snippets у видачі.

Результат

Процес публікації та первинної On-Page оптимізації статті скоротився з 15-20 хвилин ручної роботи до 5 секунд автоматичного виконання скрипта.
